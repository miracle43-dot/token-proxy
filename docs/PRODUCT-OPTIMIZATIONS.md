### 1.3 落地页优化建议（优先级排序）

**P0 - 必须修复**
1. **修复空链**："文档"和"关于我们"等 Footer 链接目前是 `href="#"`，必须填充真实页面或临时删除
2. **统一币种表述**：Footer CTA 写"$5"，按钮写"10元"，造成用户困惑，统一为一种表达
3. **补充 SLA/可用率承诺**：至少写"99.5% 可用率，详情见状态页"，并制作 `/status` 页面

**P1 - 高优先级**
4. **增加对比官方价格的折扣标注**：在价格表旁加一列"vs 官方"，直接写"官方价的 80%"，这是最强转化点
5. **增加免费试用说明**：把"注册即送体验金"升级为明确的免费额度说明（额度多少？是否需要充值？有效期多久？）
6. **扩充功能特色**：当前只有3个卡片，建议增加到 6-8 个，覆盖：智能路由、自动重试、用量监控、Key 管理、多端点切换等

**P2 - 中优先级**
7. **增加支付方式展示**：在 Pricing 区域下方加"支付方式"小节：微信、支付宝、银行卡
8. **FAQ 扩充到 8-10 条**：增加关于充值退款、额度过期、安全性、API限制等常见问题
9. **增加用户证言/案例**：即使没有真实用户，也可用"内测用户"名义放2-3条具体评价
10. **社会证明数据核实**：如果 12,000+ 不是真实数据，应删除或替换为真实数字，否则影响信任

**P3 - 低优先级（长期）**
11. **多语言切换**：英文版落地页（面向全球开发者）
12. **制作真实 Playground**：当前是静态截图，可做成真实可交互的 API 测试工具（参考 Postman/Stepfun）
13. **增加"与代理X对比"矩阵**：针对用户可能同时考虑的其他国内代理平台，做功能对比表

## 二、注册流程优化

### 2.1 当前流程

```
邮箱 + 密码 + 确认密码 → 注册 → Dashboard
```

### 2.2 问题清单

| 问题 | 影响 |
|------|------|
| 仅支持邮箱密码注册，无第三方OAuth | 转化损失约 20-30%（用户懒得填密码） |
| 无邮箱验证步骤 | 垃圾账户风险，无法发送找回密码等邮件 |
| 注册成功直接跳转 Dashboard，无引导 | 用户进来后不知道该做什么 |
| 密码仅校验"至少6位"，无强度要求 | 安全隐患 |
| 无注册协议勾选 | 法律合规风险 |
| "赠送10元体验金"无说明：是否有有效期？是否能提现？ | 用户疑虑 |

### 2.3 注册优化建议

**P0**
1. **增加邮箱验证**：注册后发验证邮件，验证通过后才能调用 API；或至少在调用 API 前强制验证
2. **注册成功后增加引导流程（Onboarding）**：
   - Step 1：创建一个 API Key（高亮按钮"创建你的第一个 Key"）
   - Step 2：获得 $5 体验额度提示
   - Step 3：复制一段示例代码开始调用
   - 参考 OpenRouter 的 New User Checklist 弹窗

**P1**
3. **增加第三方登录**：Google / GitHub OAuth（推荐优先，开发者群体覆盖率高）
4. **密码强度提示**：输入时实时显示密码强度（弱/中/强）
5. **明确体验金规则**：加 tooltip 或弹窗说明：额度有效期30天，不可提现，仅用于 API 调用

**P2**
6. **增加注册协议勾选框**：符合法规要求
7. **显示实际到账金额**：注册成功后弹窗"已到账 ¥10 ≈ $1.38 USD 体验额度"

## 三、核心卖点强化

### 3.1 当前卖点分析

现有3个卖点过于基础，且描述缺乏差异化。

| 现有卖点 | 问题 |
|----------|------|
| "无需 VPN" | 这是事实，但不是用户痛点——用户要的是"稳定"不是"方便" |
| "99.9% 可用率" | 没有实际承诺（页面没写 SLA 数据） |
| "透明定价" | 太抽象，用户需要的是"比官方便宜 XX%" |

### 3.2 卖点强化方向

**最强卖点应该是：价格折扣 + 稳定性的组合**

建议新的卖点矩阵：

| 卖点 | 描述（建议） | 优先级 |
|------|-------------|:------:|
| **价格折扣** | "GPT-4o 官方价 8 折，Claude 3.5 Sonnet 9 折" | P0 |
| **稳定直连** | "国内多节点冗余，99.5%+ 可用率，自动故障转移" | P0 |
| **秒级响应** | "平均响应延迟 < 200ms（对比官方+代理延迟数据）" | P1 |
| **智能路由** | "自动选择最低价模型完成任务，节省高达 40% 成本" | P1 |
| **即开即用** | "注册 30 秒，获得 Key，立即调用，无需审核" | P1 |
| **用量洞察** | "实时 Token 消耗追踪，设置预算警报，避免超支" | P2 |
| **多模型统一** | "一个 Key，访问 OpenAI / Claude / Gemini / 百度/阿里所有模型" | P2 |
| **合规安全** | "数据不留存，支持企业合规需求" | P3 |

### 3.3 关键建议：把折扣写进 Hero 区

Hero 副标题建议从：
> "一个 API Key，访问所有主流大模型。稳定、简单、无隐藏费用。"

改为：
> "**GPT-4o 仅官方价 8 折**，Claude / Gemini 同步折扣，一个 Key 接入全球所有模型。无需 VPN，注册即送体验金。"

这是转化率最高的改动。

## 四、用户留存功能

### 4.1 当前 Dashboard 功能（从文件结构推断）

- API Keys 管理
- 用量查看（Usage）
- 充值（Recharge）
- Playground（在线调试）
- 定价表

### 4.2 留存功能缺口

**当前产品缺少的关键留存功能：**

| 功能 | 为什么重要 | 优先级 |
|------|-----------|:------:|
| **预算/额度警报** | 防止用户意外超支，是用户的核心焦虑 | P0 |
| **用量统计面板** | 让用户看到自己花了多少钱、用量趋势，需要比"今日12,847 tokens"更详细 | P0 |
| **Key 使用分析** | 哪个 Key 用量最大？调用了哪些模型？帮助用户优化使用 | P1 |
| **充值优惠/套餐包** | 除了按量付费，可推出"充100送10"等套餐包，提升客单价 | P1 |
| **团队协作/子账号** | 企业用户需要多人共享额度，这是高价值用户的核心需求 | P2 |
| **Webhook / 用量通知** | 通过飞书/微信/邮件推送余额不足警报 | P2 |
| **API 日志/请求记录** | 用户可查看历史请求详情，用于调试和计费核查 | P2 |
| **模型推荐/成本优化建议** | "您的请求中 60% 可切换到 gpt-4o-mini，节省约 $12/月" | P3 |
| **OpenAI 兼容 SDK** | 提供 Python/JS SDK，一行代码迁移，参考 OpenRouter | P3 |
| **落地页增加文档入口** | 完整 API 文档 + 集成指南（LangChain、AutoGPT 等） | P1 |

### 4.3 留存优先级建议

**短期（1-2个月）：**

1. **用量仪表盘升级**：在 Dashboard 增加图表（折线图/柱状图），展示7日/30日用量趋势，以及各模型费用占比饼图
2. **预算警报**：用户可设置"余额低于 ¥20 时发送邮件/飞书通知"
3. **完善 Playground**：做成真实可交互的 API 测试工具（目前是静态截图）

**中期（3-6个月）：**

4. **充值套餐**：推出"首充双倍"、"月套餐"等，提升 LTV
5. **Webhook 通知**：对接飞书机器人，实时推送用量警报
6. **SDK 文档**：提供 Python/Node.js SDK，参考 OpenAI 官方 SDK 用法，降低迁移成本

**长期（6个月+）：**

7. **子账号/团队管理**：支持企业用户创建子账号，设置权限和额度限制
8. **API 请求日志**：完整记录每次请求，用于计费核查和调试
