## 1. 账号定位与人设

### 账号名称建议

建议采用**「TokenProxy」**作为主账号名称，中文可配合使用**「TokenProxy官方」**或**「AI调参侠」**等昵称作为人设化名。选择理由：TokenProxy 本身是一个技术感强、易记忆的品牌名，在抖音和小红书的搜索场景中，用户搜索"Token"或"API"相关关键词时更容易触达。若运营方希望更亲民，也可考虑使用**「AI便利店」**或**「API小卖部」**这类更口语化的名称，降低技术内容给普通开发者的心理门槛。

### 人设建议

建议打造**「踩坑老手 + 实操型技术博主」**人设，而非高高在上的专家形象。具体设定如下：

**人设名称**：阿睿（ARui）或 小P（取 Proxy 首字母）

**人物背景**：93年生，曾在大厂做过后端开发，后来自己出来做独立开发者和AI应用创业者。有真实的开发项目经验，能站在开发者立场分享，而非纯销售导向。内容中会经常出现"我之前踩过这个坑"、"这个坑花了我3天才排查出来"等真实经历分享，让人设可信度高。

**说话风格**：语速适中，视频表达偏口语化，像在跟朋友聊天。会用"兄弟们"、"这个真的很重要"、"别踩我的坑"等口头禅。每期视频结尾固定说"有问题评论区见，我是阿睿，我们下期见"。评论区会认真回复每一条问题，建立信任感。

**差异化优势**：不只讲概念，更讲实操。每一个内容背后都有可以落地执行的步骤，用户看完真的能跑通一个API调用、搭建一个小项目。

### 目标用户画像

核心用户分为三类，每类用户的内容触达策略略有不同：

**第一类：个人独立开发者（占比约50%）**
年龄集中在22-35岁，有一定编程基础（至少会Python/JavaScript），正在做自己的AI应用或副业项目。典型特征是：有GitHub账号、逛掘金/StackOverflow、在技术群里活跃。他们的核心痛点是：没有海外信用卡，无法开通OpenAI账号；充值麻烦，需要找代充或者自己搭建代理；担心充值渠道安全性。他们的典型问题包括"国内怎么用ChatGPT API"、"没有外币信用卡怎么调用GPT-4"等。

**第二类：小型创业团队/工作室（占比约35%）**
3-10人规模，正在开发AI相关产品（如AI客服、AI写作工具、AI图像生成等）。核心痛点是：团队管理混乱，API Key散落在各个成员手中，无法统一计费；海外支付受阻，项目进度被卡脖子；想要更低成本的API来源但没有渠道。这部分用户的决策链路较长，通常需要对比多家供应商的稳定性和价格。

**第三类：AI应用学习者和爱好者（占比约15%）**
年龄跨度更大，从高中生到35岁职场人都有。可能是看了某篇爆款文章后来搜索API相关内容，或者想用AI做点小工具自用。学习者没有付费习惯，但能带来曝光和传播，是未来的潜在付费用户。内容对他们要足够简单，降低使用门槛，用"3分钟上手"等钩子吸引关注。

**一句话价值主张（品牌slogan）**：
> "国内开发者的AI API直通车——充值即用，不限流量，合规稳定"

**三条细分价值主张（对应三类用户）**：

对学习者/爱好者：**"注册就送10元体验金，够你跑100次GPT-3.5。门槛最低的AI调用入口，5分钟搭建你的第一个AI应用。"** 用低成本试用策略吸引他们入门。

## 2. 内容选题（至少10个）

### 选题1：注册到调通API——零基础教学

**选题标题**："没有信用卡？3分钟拿到你的第一个AI API Key【TokenProxy教程】"

**内容方向**：这是一期纯新手向的step-by-step教学视频。从访问TokenProxy官网开始，演示注册账号（强调不需要手机号、不需要信用卡）、充值余额（强调支持微信/支付宝）、创建API Key、到用Python写第一行调用代码，完整跑通ChatGPT的对话接口。全程录屏，画面左下角显示正在操作的浏览器，右下角显示终端输出结果。结尾告诉用户充值套餐和价格表在哪里查看。

**预计时长/形式**：抖音8-10分钟（完播率对新手友好），小红书图文帖（截图+注释，分8个步骤）。竖屏录制，出镜画中画展示人设。

**预计引流效果**：引流效果高，搜索"ChatGPT API 国内"等关键词的流量精准，用户进来后目的明确就是"要用API"，转化率高。预估单条播放量3000-10000，评论转化率5-8%。

### 选题2：竞品对比——为什么选TokenProxy而不是硅基流动

**选题标题**："TokenProxy vs 硅基流动——实测对比告诉你谁更值得用"

**内容方向**：选取两家平台，分别用同一模型（GPT-4o）跑10次相同的prompt，记录响应时间、成功率和单价，整理成表格对比。内容不做拉踩，客观呈现数据差异。如果TokenProxy在某些维度有优势（如响应速度、价格、界面体验），在数据基础上强调；如果某方面不如竞品，诚实带过但强调差异化价值（如客服响应速度、充值便利性）。

**预计时长/形式**：抖音12-15分钟（需要讲数据），小红书表格对比图（配少量文字说明）。竖屏，PPT式数据可视化。

**预计引流效果**：高冲突性选题，竞品对比类内容在技术圈有天然传播性，容易引发讨论和转发。预估播放量5000-20000，但评论区会有人质疑，需准备好应对话术。

### 选题4：我的API账单为什么这么贵？

**选题标题**："AI API每个月花我1000块——我是怎么排查和优化的"

**内容方向**：以博主自己的真实经历为素材，讲述某个月API账单突然暴涨的经历（从每月300元飙到1000元），然后展示排查过程：查看TokenProxy的用量明细，发现是某个项目用了GPT-4o但其实GPT-4o-mini就够了；是max_tokens设置过大导致输出token浪费；是没有开缓存导致重复请求。最后给出4条优化建议：选对模型、合理设置max_tokens、用流式输出避免超时重试、在客户端做请求去重。结尾预告下期内容"怎么用最小成本跑完整个开发周期"。

**预计时长/形式**：抖音10-12分钟（需要展示后台数据截图），小红书长图（截取用量明细表格，逐条分析）。竖屏，出镜讲述+屏幕录制结合。

**预计引流效果**：真实经历分享类内容信任度高，能引发评论区共鸣"我也是这样"。预估播放量3000-8000，但评论区和收藏率高，适合沉淀为长期搜索流量。

### 选题9：API常见报错合集——附解决方案

**选题标题**："API报错401/402/429/500？这可能是全网最全的排查指南"

**内容方向**：整理TokenProxy使用过程中最常见的4种错误码，分别演示每个错误的含义、出现原因和解决方法。401 Unauthorized（Key无效或已删除）、402 Payment Required（余额不足）、429 Rate Limit（请求过快，限流）、500 Internal Server Error（上游服务挂了）。每个错误用实际演示的方式展示，结尾告诉用户"遇到其他报错加我主页客服微信"。

**预计时长/形式**：抖音8-10分钟（分4段，每段2分钟讲一个错误），小红书图文帖（4张错误对照图）。竖屏，屏幕录制为主。

**预计引流效果**：搜索型内容，用户遇到问题后会主动搜索对应错误码，属于被动引流。预估单条播放量1500-5000，但精准度高，转化率接近10-15%。

### 选题10：TokenProxy vs 自建代理——哪个更适合小团队

**选题标题**："自己搭建API代理？还是直接用TokenProxy？我算了笔账给你看"

**内容方向**：从成本和技术门槛两个维度对比自建代理和直接使用TokenProxy。自建方案成本拆解：需要境外服务器（每月$20-50）+ 技术人力（至少要1个会运维的人）+ 时间成本（搭建调试需3-7天）+ 风险成本（IP被封、代理不稳定）。TokenProxy成本：充值直接用，无技术门槛，0运维成本。结论：除非调用量极大（每月tokens数亿级别），否则TokenProxy性价比更高。适合人群：5人以下的创业团队、个人开发者。

**预计时长/形式**：抖音10-12分钟（需要讲逻辑），小红书对比图（两列对比，配数字）。竖屏，出镜+白板演示结合。

**预计引流效果**：决策型内容，用户在评估阶段会反复看这类对比，对最终转化有重要作用。预估播放量3000-10000。

## 4. 平台差异化策略

### 抖音 vs 小红书用户差异

理解两个平台用户群体的根本差异是制定差异化策略的前提。抖音和小红书虽然都是内容平台，但用户的使用场景、心智状态和内容偏好存在显著差异。

**抖音用户特征**：
抖音的用户基数更大、年龄层次更广，但核心用户群体集中在18-35岁。抖音是一个「泛娱乐+算法推荐」平台，用户打开抖音的初始心理状态是"放松和消费内容"，不是"主动学习和解决问题"。这意味着抖音用户更容易被情绪、故事和即时价值吸引，但耐心有限，超过3分钟的内容如果没有强钩子容易流失。抖音的流量分发高度依赖算法，账号粉丝量不等于内容曝光量，每条内容都需要独立跑出推荐。抖音用户的决策链路短，好内容能在短时间内完成"种草→搜索→转化"的全链路。

**小红书用户特征**：
小红书的用户以18-30岁女性为主（男生比例在AI/科技内容方向会更高，可达40%），整体用户画像偏向"有消费决策需求的年轻人"。小红书是一个「搜索+种草」平台，用户打开小红书时往往带着问题或明确需求（"这个东西值不值得买"、"这个工具怎么用"），属于半主动内容消费状态。这使得小红书的搜索流量占比高（很多内容靠关键词被长期搜索到），内容生命周期长，一篇好的教程可以持续几个月甚至几年带来自然流量。小红书用户更注重内容的"可操作性"和"步骤完整性"，偏爱图文教程，评论区常见"码住"、"谢谢博主"等互动。

**目标用户在两平台的行为差异**：

| 维度 | 抖音 | 小红书 |
|------|------|--------|
| 打开目的 | 娱乐放松 | 找答案/种草 |
| 黄金时长 | 15-60秒（泛娱乐）/ 3-10分钟（教程） | 无限制（主动浏览搜索） |
| 内容偏好 | 故事性、娱乐性、冲突感 | 实用性、步骤完整、有图有真相 |
| 搜索主动性 | 低，通常算法推什么看什么 | 高，带着问题来 |
| 转化路径 | 视频→评论区→主页→转化 | 搜索→图文→收藏→转化 |
| 互动方式 | 点赞、评论、关注 | 收藏、评论、保存图片 |

### 各自的重点内容类型

基于平台差异，两平台的内容重心应有明显区分：

**抖音主打内容类型**：

第一类是「即时满足型」内容，时长控制在30-60秒，核心是"一句话说清楚一个技巧"，例如"API Key这样设置才安全"、"余额不足时系统会返回这个错误码"。这类内容在抖音能获得较高完播率和转发率，但引流效果有限，主要作用是积累粉丝和建立人设。

第二类是「实操教程型」内容，时长6-15分钟，完整展示一个操作全流程，例如"5分钟搭建AI客服"、"Cursor配置TokenProxy"。这类内容是抖音的主要流量来源，需要在视频前3秒设置强钩子（"没有信用卡也能用GPT-4，方法在这里"），中间保持节奏紧凑，避免长时间无操作的纯屏幕录制。

第三类是「数据测评型」内容，用真实数据说话，例如"TokenProxy vs 硅基流动，我跑了100次实测对比"，这类内容在抖音能引发讨论，适合在评论区引导用户发表观点，提高互动率。

**小红书主打内容类型**：

第一类是「保姆级图文教程」，6-10张图，每张图附带详细注释和截图，目标是"让完全不懂的小白也能照着做成功"。这类内容是小红书搜索流量的主力，也是TokenProxy获客的最有效内容类型。典型结构：封面图（痛点标题）→ 第一步注册账号 → 第二步充值 → 第三步创建Key → 第四步调用测试 → 结尾引流卡片。

第二类是「对比测评表格」，以表格形式横向对比多个产品或方案，一目了然。例如"GPT-4o vs Claude 3.5 vs Gemini 1.5 价格速度实测对比"，用表格呈现数据比文字描述更有说服力，也更容易被收藏。

第三类是「避坑指南」类图文，标题带有"注意"、"千万别"、"这个坑"等字眼，命中用户搜索痛点。例如"API调用最常见的5个错误及解决方案"，这类内容搜索精准、生命周期长，能持续带来自然流量。

**内容分配的底层逻辑**：
抖音承担的是「曝光+信任建立」，让更多人知道TokenProxy这个平台，在用户心中建立"做AI开发可以用TokenProxy"的认知，但不追求在抖音直接完成转化。小红书承担的是「转化+长尾获客」，用户在小红书搜索到详细教程后，直接跳转官网注册的概率远高于抖音。内容规划时，每条抖音内容都应设计"钩子"引导用户去小红书搜索更详细的教程，而小红书教程底部统一加"有疑问欢迎评论，我帮你解答 + 官网链接"。

## 5. 引流链路

### 从内容到网站的完整路径

整个引流链路分为四个层级，每一层级都有明确的用户行为目标和优化重点：

**第一层级：触达与吸引**
用户刷到我们的内容（抖音算法推荐 or 小红书搜索）。这个层级的关键是视频/图文的前3秒能否抓住用户注意力。对于抖音，算法考核的核心指标是完播率和互动率，因此视频开头必须设置强钩子（例如"没有信用卡也能用ChatGPT API，方法很简单"）；对于小红书，算法考核的是收藏率和搜索转化，因此封面图和标题必须精准命中用户搜索词。

**第二层级：激发兴趣与建立信任**
用户看完内容后产生"想试试"的念头，但没有立刻离开。这个层级的关键是内容能否回答用户心里的两个问题："这个真的能用吗"和"会不会很复杂"。解决方案：视频中实时演示操作全过程（不剪辑中间步骤），展示真实充值和API调用结果；评论区置顶"已验证可用"的反馈，增强社会证明。

**第三层级：引导行动**
用户决定去TokenProxy官网看看。这个层级的关键是行动路径要短、要清晰。抖音视频结尾固定口播"官网 tokenproxy.com，链接在评论区置顶"，配合屏幕录制展示浏览器地址栏；小红书图文在最后一张图放"如何找到TokenProxy"的截图指引，配合文字"点击我的头像，简介里有官网链接"。行动指令要具体，不要说"大家去看看"，要说"现在打开浏览器，输入tokenproxy.com，注册账号充值10元，就能跑通第一个API"。

**第四层级：完成注册与首次充值**
用户到达TokenProxy官网后，需要尽可能降低首次使用的摩擦。关键优化点：注册流程不超过2分钟（邮箱+密码，无需手机验证）；新用户首次充值提供¥10体验套餐；注册成功后立刻展示"下一步"引导——创建第一个API Key，并在Key创建页面提供"测试调用"的示例代码，让用户能立即验证Key是否可用。

**全链路漏斗优化指标**：

| 链路层级 | 核心指标 | 健康值 |
|----------|----------|--------|
| 触达 | 播放量/曝光量 | 基准2000 |
| 吸引 | 完播率（抖音）/收藏率（小红书） | 完播率>30%，收藏率>5% |
| 引导 | 评论/私信中的网址访问请求量 | 每周>50 |
| 转化 | 官网注册→首充转化率 | >15% |

### 如何让用户记住网站

让用户记住一个陌生的网站域名是极具挑战的任务，TokenProxy这个域名本身有一定优势（Proxy好记），但仍需要通过多种手段强化记忆：

**重复曝光法**：每期内容至少出现3次网站名称——开头介绍"我是TokenProxy官方账号"、中期演示"打开tokenproxy.com"、结尾"有问题去官网找客服"。通过多次重复让用户形成条件反射。

**谐音/口诀记忆法**：在口播中穿插"tokenproxy，记住了吗，就是Token的Proxy，代理你的AI调用"这样的解释，帮助用户通过语音强化记忆。或者设计简短的口号"TokenProxy，AI调用更容易"在每期结尾重复。

**视觉锚点法**：在所有视频的右下角固定放置TokenProxy的水印logo，在所有图片的固定位置放置品牌色（#7C3AED紫色）作为视觉识别元素。用户在多次观看后会把紫色和TokenProxy品牌关联起来。

**场景联想记忆法**：在内容中植入"当你需要用API的时候，就去tokenproxy.com"这样的场景提示，建立"需求触发→想起TokenProxy"的反射。例如在选题9（API报错合集）的每种错误讲解后都说："遇到这个问题不要慌，去tokenproxy.com找客服，或者看看帮助文档"。

**域名优势利用**：TokenProxy域名本身具有描述性——"Token代理"，在教程内容中解释这个含义（"Proxy就是代理，帮你把请求转发到OpenAI的服务器"），让用户觉得这个域名"有意义"而不是"随机字符串"，记住的概率更高。

### 钩子话术设计

钩子是引流内容的核心，一套有效的钩子话术需要同时满足"引起注意"和"激发行动"两个条件。以下是针对不同场景设计的钩子话术：

**引流文案模板（在内容结尾固定使用）**：
