## 1. 我们是谁

### 1.1 TokenProxy 定位

TokenProxy 是**面向中国小B用户的 AI API 代理销售平台**。

用户注册 → 充值 → 获得 API Key → 调用大模型 API（豆包、OpenAI、Claude等）→ 我们赚取 token 差价

我们的核心定位：**帮火山引擎把豆包大模型 API 卖给千千万万小微企业、夫妻店、技术开发者**，我们是火山引擎的"最后一公里"渠道。

### 1.2 现有能力（产品 / 技术 / 服务）

| 能力维度 | 详情 |
|---------|------|
| **产品** | TokenProxy 代理平台（Vue3 前端 + Node.js 后端 + SQLite），已跑通 MVP，注册 → 充值 → 调用全流程 |
| **技术** | OpenAI 兼容接口（`/v1/chat/completions`），支持流式输出，支持多通道熔断切换 |
| **部署** | 支持一键部署在全球主流平台（Railway/阿里云/腾讯云） |
| **运营** | 已有真实付费用户跑在平台上，有完整用户管理体系 |
| **团队** | 全栈开发 + 运营 + 内容团队，具备快速起量能力 |

### 1.3 已有资源

- **现成平台**：MVP 已上线，可立即接入豆包 API
- **用户基础**：已有自然注册用户，具备冷启动条件
- **内容矩阵**：抖音/小红书 AI 技术内容账号，具备教育市场和获客能力
- **老板关系**：认识豆包/火山引擎内部人，沟通成本低，信任度高

### 2.1 国内 AI API 市场现状

| 年增速 | 60-80% |
| 中小开发者/小B占比 | ~40%（金额口径） |

**主要玩家价格对比（2026年4月）**

| 模型 | 输入价格（元/千token） | 输出价格（元/千token） |
|------|----------------------|----------------------|
| OpenAI GPT-4o | ~7.5（美元换算） | ~30（美元换算） |

> 豆包的价格是 OpenAI 的 **5-50分之一**，价格竞争力极强。

### 2.2 小B用户（夫妻店 / 小微企业）的痛点

我们观察到的小B用户真实痛点：

| 痛点 | 具体描述 |
|------|---------|
| **支付壁垒** | 没法开通信用卡，无法充值 OpenAI；市面上代充值跑路风险高 |
| **价格太贵** | 小B用AI做客服机器人/文案生成/自动化，每天几十元，一年就是几千元，吃不消 |
| **不懂技术** | 不知道什么是 API Key，不懂 Python / Node.js，需要"傻瓜式"引导 |
| **没有发票** | 个体户/夫妻店需要发票报销，大平台服务难以开具 |
| **充值门槛高** | 最低充值金额太高，用不完浪费；没有小额度试用 |

**本质问题**：大厂追求大客户，小B是"被遗忘的角落"。

### 2.3 为什么大厂不愿意做这一块

| 大厂逻辑 | 实际情况 |
|---------|---------|
| 大厂追求大客户（KA），客单价高，sales effort 划算 | 小B客单价低（100-500元/月），大厂sales懒得跑 |
| 大厂要标准化产品，不愿意定制 | 小B需要手把手服务，大厂服务不过来 |
| 大厂要合规、合同、发票，对小B来说流程太重 | 夫妻店老板"要发票，但不想签合同" |
| 大厂定价美元计价，有汇率和监管风险 | 小B用户对价格敏感，人民币计价才是刚需 |

**结论**：豆包价格足够低 + 我们服务足够近 = 完美互补。我们做脏活累活，火山引擎供弹药。

## 3. 目标客户

### 3.1 客户画像

**核心客户：夫妻店 / 小微企业老板 + 技术开发者**

**画像一：小微商家（夫妻店）**

- 典型场景：奶茶店、花店、打印店、小饭馆、服装店
- 规模：1-5人，不需要员工
- 用 AI 做什么：
  - 用 AI 自动回复顾客微信（智能客服）
  - 用 AI 生成朋友圈文案、营销海报文字
  - 用 AI 做小红书/大众点评运营文案
- 月消费能力：**50-300元/月**
- 决策链：老板自己说了算，不需要开会

**画像二：独立开发者 / 技术个体户**

- 典型场景：接私活、做小程序、卖插件、做 SaaS 小工具
- 规模：1-3人，个人创业者为主
- 用 AI 做什么：
  - 调用大模型 API 做 AI 应用（聊天机器人、AI写作工具）
  - 做微信/抖音小程序的 AI 功能
- 月消费能力：**100-800元/月**
- 决策链：自己决定，付费爽快

**画像三：小型创业团队**

- 典型场景：10人以下的 AI 应用创业团队
- 用 AI 做什么：产品原型、快速验证 AI 功能
- 月消费能力：**500-3000元/月**

## 4. 商业模式

### 4.1 核心逻辑：火山引擎渠道代理

```
火山引擎（豆包API）
        ↓ 给我们折扣价（如挂牌价 5-7 折）
TokenProxy（我们）
        ↓ 加价卖（保留 15-30% 毛利）
小B用户（夫妻店 / 开发者）
```

我们不碰模型底层，只做：**渠道 + 封装 + 服务**。

### 4.2 收入来源

| 收入来源 | 说明 | 预估毛利贡献 |
|---------|------|------------|
| **Token 差价** | 火山引擎给我们的价格 vs 我们卖给用户的价格差 | ~60-70% |
| **部署/开户费** | 企业用户定制部署，一次性收取 | ~10-15% |
| **服务费** | 培训、技术支持、定制开发 | ~15-25% |
| **充值套餐溢价** | 套餐包销售，批量采购降低单位成本 | ~5-10% |

### 4.3 具体分钱机制

**场景示例（以299元/月专业版套餐为例）**

| 项目 | 金额 |
|------|------|
| 用户支付 | 299元/月 |
| 豆包API成本（~1000万token） | ~120元（按火山引擎折扣价，约12元/百万token） |
| TokenProxy毛利 | **179元（60%毛利率）** |

**规模化后的月收入估算**

| 用户规模 | 月收入 | 年收入 |
|---------|--------|--------|
| 100个付费用户 | ~3万元 | ~36万元 |
| 500个付费用户 | ~15万元 | ~180万元 |
| 2000个付费用户 | ~60万元 | ~720万元 |

## 6. 我们需要什么

### 6.1 代理价格（折扣条件）

**核心诉求：以有竞争力的折扣价格接入豆包API**

| 条件 | 诉求 |
|------|------|
| **折扣比例** | 建议：**挂牌价 5-7折**（即 5折拿货，卖给用户保持60%+毛利） |
| **预付模式** | 按调用量月结，或预付费充值（类似云厂商模式） |
| **价格保护** | 签约期内不涨价，或涨价需提前3个月通知 |
| **专属折扣码** | 给TokenProxy用户专属价格区隔，避免价格体系混乱 |
| **最低消费** | 暂无最低消费要求，希望从0开始培养 |

**参考依据**：
- 硅基流动给代理的折扣约为 6-8 折
- 火山引擎云服务渠道代理通常为 7-8 折
- 我们作为专注小B的垂直渠道，建议争取 **5-6 折**

### 6.2 技术支持

| 需求 | 说明 |
|------|------|
| **API 稳定性** | SLA ≥ 99.5%，故障需有快速响应通道 |
| **专属技术支持群** | 对接火山引擎技术负责人，故障时优先处理 |
| **新模型优先体验** | 豆包新模型上线，我们第一时间接入并推广 |
| **计费对账透明** | 提供详细的 API 调用账单，支持对账核查 |
| **沙箱/测试环境** | 免费沙箱环境，方便我们测试接入 |

### 6.3 商务支持

| 需求 | 说明 |
|------|------|
| **官方背书** | 将 TokenProxy 列为"豆包API授权渠道合作伙伴"（可官网公示） |
| **联合营销** | 联合举办小B商家AI培训、线上直播等 |
| **客户推荐** | 火山引擎销售团队遇到小B客户时，推荐转介绍给我们 |

## 7. 合作方案建议

### 7.1 推荐合作形式

**推荐：渠道代理商合作（轻量级，先跑通再深化）**

| 项目 | 内容 |
|------|------|
| **合作模式** | 渠道代理：我们作为火山引擎授权渠道商销售豆包API |
| **合作门槛** | 低门槛切入，先签框架协议，不要求年保底 |
| **账期** | 月结（次月15日前结清上月API调用费） |
| **授权范围** | 线上销售（网站/App/小程序），可开代理资质证明 |

### 7.2 第一步怎么走

**Phase 0（第1-2周）：技术对接**
- 双方技术对接，TokenProxy 接入豆包 API（非正式环境）
- 完成计费逻辑对接、回调机制确认
- 目标：跑通"注册→充值→调用豆包API→扣费"全流程

**Phase 1（第3-4周）：小范围灰度**
- 内部员工 + 种子用户（50-100人）灰度测试
- 收集反馈，优化定价和用户体验
- 目标：验证产品体验，输出测试报告

**Phase 2（第2-3月）：正式上线 + 推广**
- 官方发布：TokenProxy 豆包API代理正式上线
- 抖音/小红书同步发布推广内容
- 目标：首批 500 付费用户，月流水 5-10 万元

### 7.3 近期目标（3个月 / 6个月 / 12个月）

| 时间节点 | 核心目标 | 关键指标 |
|---------|---------|---------|
| **3个月** | 跑通全流程，小规模验证 | 付费用户 ≥ 500；月流水 ≥ 5万元；豆包API日调用量 ≥ 1亿tokens |
| **6个月** | 稳定运营，建立口碑 | 付费用户 ≥ 2,000；月流水 ≥ 20万元；续费率 ≥ 70% |
| **12个月** | 规模化增长，复制打法 | 付费用户 ≥ 10,000；月流水 ≥ 100万元；成为豆包API头部渠道 |

## 附录：竞争壁垒说明

TokenProxy 的核心壁垒不是技术，而是**渠道和服务**：

1. **信任壁垒**：小B商家更愿意相信"看得见摸得着"的微信群服务，而不是网页表单
2. **内容壁垒**：抖音/小红书积累的粉丝，是低成本的精准获客渠道
3. **习惯壁垒**：用户一旦习惯了我们的操作界面，换平台的动力很低
4. **规模壁垒**：用户量越大，API采购量越大，拿到更低的折扣，形成正循环
