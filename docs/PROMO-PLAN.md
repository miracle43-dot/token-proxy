## 三、新用户注册赠送方案

### 3.1 赠送内容：**注册送 ¥10 体验余额**

**为什么送余额而不是 Token？**
- Token 价格波动大，送固定数量用户感知价值不稳定
- 送余额 = 送购买力，用户可以自主选择购买任意模型
- 余额不会过期（建议设置 30 天有效期，催促激活）

**门槛设计：**
- 仅限新用户（手机号/邮箱未注册）
- 注册后自动发放，无需人工审核
- 余额仅限抵扣，不可提现

### 3.2 升级激励：**首单充值 1 元 → 再送 ¥20 余额**

**目的**：从「注册」推进到「付费行为」，筛选真正有需求的用户。

**规则**：
- 用户完成首次充值（任意金额 ≥ ¥1）
- 系统自动赠送 ¥20 余额到账户
- 赠送余额可用于后续任意消费

### 3.3 力度评估

| 阶段 | 成本 | 目的 |
|------|------|------|
| 注册即送 ¥10 | 获客成本低，测试注册转化 | 降低首次使用门槛 |
| 首充1元再送 ¥20 | 筛选真实付费意愿用户 | 建立付费习惯 |

**风险控制**：
- 每个用户最多获得 ¥30（注册 ¥10 + 首充 ¥20）
- 赠送余额仅限 AI API 消费，不可提现，实质成本远低于账面金额
- 设置防刷机制：同一 IP / 设备 / 手机号仅限参与一次

## 四、邀请返利方案（老带新）

### 4.1 机制设计

| 角色 | 奖励 |
|------|------|
| 邀请人（老用户） | 被邀请人每笔消费 → 邀请人获得 10% 返现（长期有效） |
| 被邀请人（新用户） | 额外获得 ¥5 邀请奖励（叠加注册赠送） |

### 4.2 为什么做 10% 返现而非一次性奖励？
- **长期激励**：持续刺激老用户主动分享（分享一次，长期收益）
- **绑定消费**：老用户希望被邀请人持续消费，自己才能持续返现
- **对平台友好**：本质是佣金分成，只要用户持续付费，平台始终盈利

### 4.3 邀请码机制
- 每个用户有唯一邀请码
- 新用户注册时填写邀请码，双方同时获得奖励
- 在用户余额页面、个人中心显眼位置展示邀请码和已返现金额

## 十、总结

本次活动的核心逻辑：**「低门槛进入 + 筛选付费 + 长期绑定」**

1. **注册送 ¥10**：让用户零成本体验产品
2. **首充 ¥1 再送 ¥20**：驱动第一次付费行为，验证付费意愿
3. **邀请 10% 返现**：建立老带新飞轮，降低长期获客成本
4. **限时限量**：制造紧迫感，加速决策

> 原则：每一笔赠送都指向「下一次消费」，而不是单纯的补贴。补贴只是手段，验证商业模式才是目的。
