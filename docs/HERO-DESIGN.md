# Hero 区设计规范

> 本文档定义 token-proxy 首页 Hero 区域的设计系统，作为迭代参考标准。

---

## 1. 设计定位

- **产品定位**：中文市场稳定低价 AI API 代理，面向开发者和技术爱好者
- **用户感受**：专业、可信赖、购买流程简单
- **核心关键词**：稳定 · 低价 · 快速 · 国内直连


| 场景 | 数值 |
|------|------|
| 桌面 Section 内边距 | 64px |
| 平板 (≤1024px) | 48px |
| 移动 (≤768px) | 24px |

| 断点 | 宽度 | 说明 |
|------|------|------|
| Desktop | > 1024px | 完整布局 |
| Tablet | 768px - 1024px | 两列压缩 |
| Mobile | < 768px | 单列堆叠 |
| Small Mobile | < 480px | 极小屏优化 |

### 间距系统（8px 基准）
```
4px  / 8px / 12px / 16px / 24px / 32px / 40px / 48px / 64px / 80px / 120px
```

## 7. 组件规范

### 按钮
- **Primary**：`background: #7C3AED; border-radius: 8px; padding: 10px 20px`
- **Ghost**：`border: 1.5px solid #7C3AED; border-radius: 8px`
- **大按钮**：padding 16px 32px，字号 16px
- **特大按钮**：padding 18px 40px，字号 16px

### 圆角
| 组件 | 圆角值 |
|------|--------|
| 按钮 | 8px |
| 卡片 | 12px |
| 代码窗口 | 12px |
| Model Chip 容器 | 12px（外层） |
| 头像/头像组 | 50% |

### 阴影
- 代码窗口：`box-shadow: 0 25px 50px -12px rgba(0, 0, 0, 0.25)`

---

## 8. 设计原则

1. **技术感 + 可信赖**：深色代码窗口传达专业感，绿色状态指示建立信任
2. **信息密度适中**：Hero 区左文右图，模型展示 + 代码窗口 + 三大优势三合一
3. **移动端优先降级**：移动端模型网格变为 3 列或 2 列，优势条垂直堆叠
4. **一致性**：复用现有 `--purple` 变量，保持与全站风格一致
5. **行动明确**：CTA 按钮首位突出，副按钮辅助导航
