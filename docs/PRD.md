## 1. 产品概述

### 1.1 产品定位

TokenProxy 是一个 **AI API 代理销售平台**，面向开发者提供 OpenAI、Claude、Gemini 等大模型 API 的代理转发服务。

用户注册充值 → 获得平台 API Key → 通过代理调用大模型 API → 平台赚取差价

核心价值：**让用户以更便捷的方式、更透明的价格使用大模型 API，同时平台获取利润空间。**

### 1.2 目标用户

| 用户画像 | 特征 | 痛点 |
|---------|------|------|
| 个人开发者 | 技术能力中等，用量不大 | 不会科学上网、支付困难 |
| 小型创业团队 | 需要调用大模型API | 海外支付卡限制、管理分散 |
| AI应用开发者 | 有稳定调用需求 | 想要更低成本的API来源 |

### 1.3 成功指标（KPI）

- 注册用户数
- 充值转化率（注册→充值）
- API 调用量（tokens/月）
- 毛利率

## 3. 用户流程

### 3.1 典型用户路径

```
注册账号 → 充值余额 → 创建API Key → 阅读文档 → 集成API → 正常使用 → 余额不足 → 再次充值
```

### 3.2 关键页面流程

#### 注册/登录
- 简洁的两步式：输入邮箱/密码 → 登录成功
- 无需手机号（当前MVP）
- JWT Token 7天有效

#### 充值流程（规划中）
```
选择充值金额 → 选择支付方式（微信/支付宝）→ 扫码支付 → 支付成功 → 余额到账
```

#### API 调用流程
```
POST /v1/chat/completions
Headers: Authorization: Bearer sk-proxy-xxx
Body: { model: "gpt-4o", messages: [...] }

→ 系统验证Key → 检查余额 → 转发给上游API → 解析usage → 扣除余额 → 返回响应
```

## 5. 风险与挑战

| 风险 | 应对策略 |
|------|---------|
| 上游API涨价 | 建立价格联动机制，及时调整定价 |
| 用户滥用（盗刷） | API Key 隔离 + 用量异常检测 |
| 支付合规性 | 接入正规支付渠道，留存交易记录 |
| 竞争对手压价 | 强调稳定性和服务质量 |
